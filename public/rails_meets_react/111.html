componentWillReceiveProps
This function is invoked whenever a component is receiving new props - however not for the initial
render.
It takes
nextProps
argument, which represents the props about to be set. The old props are still
available at
@props
.
In one of our projects we use the
redactor
. We render a
textarea
connected to this library.
One of the textarea’s properties is an initial value to edit by the user (
defaultValue
). However,
the
defaultValue
is not known at the very beginning of the component’s life - it is fetched via
external (from the component’s point of view) API.
defaultValue
may be updated, the good news
is - we know that it can happen only once.
To properly initialize
redactor
,
redactorSetFn
should be called as soon as we know what text
the component should be filled with.
So this is how we handle it:
componentWillReceiveProps:
(nextProps) ->
if
!
@props.defaultValue
? and
nextProps.defaultValue
?
@redactorSetFn(nextProps.defaultValue)
