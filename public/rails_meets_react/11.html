General Installation Instructions
First of all, we need to make sure we can use React in our Rails application. Hopefully, React comes
with a nice gem that integrates this library well with Rails.
react-rails
is a complete solution to
integrate and use all React features out of the box. Installation is very simple and has two steps:
1. Provide
react-rails
as a dependency in your project’s
Gemfile
:
gem
'react-rails'
2. Since React library comes in two variants -
production
, which removes debugging messages
which React typically prints to us in browser console and
development
, which have all
debug messages and exceptions available, you have to specify for all environments in your
app which one do you want. Thus, for each environment in
config/environments
, under
<YourAppName>::Application.configure do
you have to add:
config
.
react
.
variant
=
:production
# if you want production React variant
Or:
config
.
react
.
variant
=
:development
# if you want development React variant
Typically you would use
:development
variant in
config/environments/development.rb
and
config/environments/test.rb
(if you use JS test framework like Konacha) and
:production
variant in
config/environments/production.rb
.
