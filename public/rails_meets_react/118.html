Passing state
You now know how to create components, add dynamic behaviour to them with state and integrate
events and libraries from the external world. That makes you ready for creating your user interface
using React. What I want to show you in this chapter is how to avoid mistakes while composing
your UI using components.
Often the hardest part of creating React apps is distributing state in
your components.
Making this decisions wrong can make your code more complicated or make it
less performant.
There are some techniques to pass state between components tree. We’ll start with the most basic
approach - creating a
root component
which stores the whole state and distributing it across stateless
components down in its tree. Then I’ll show you how to make it more performant by using a feature
called references. Both solutions have its pros and cons - I’ll also discuss it briefly.
Root component approach
In most user interfaces you build using React you have one component’s tree per one “screen” of your
application. This architecture has one major consequence -
from the outside of the code managed
by React you usually have comfortable access only to the top-level component.
That means the
top-level component is some kind of
facade
for your user interface code.
Top-level component often is a facade to your user interface.
