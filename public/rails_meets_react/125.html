Animations
Once you have working, usable and dynamic components, animations can be the icing on the
cake. Using animation, you can: make it easier for users to notice changes; direct their attention
to something important happening; or show that things have changed due to some action by the
user.
The
Google material design guidelines
even have 4 chapters on this topic:
•
Authentic motion
•
Responsive interaction
•
Meaningful transitions
•
Delightful details
React.js provides both high-level and low-level animation APIs provided as add-ons to the core
library. We will start with the high-level
CSSTransitionGroup
which provides an easy way to
perform CSS transitions and animations when a React component is rendered, added or removed.
React.addons.CSSTransitionGroup
React.addons.CSSTransitionGroup
is a box component that takes a list of child components
that will be animated when a certain situation occurs.
There are 3 situation that can potentially trigger an animation:
• a child component is present when
CSSTransitionGroup
is rendered for the first time. You
can think about this state as “
component appeared
when page was rendered.” Animation in
this situation is indicated by the
transitionAppear
property, which is
false
by default.
• a child component is added to
CSSTransitionGroup
. Animation in this situation is indicated
by the
transitionEnter
property, which is
true
by default.
• child component is removed from
CSSTransitionGroup
. Animation in this situation is
indicated by the
transitionLeave
property, which by default is
true
.
The CSS animation is triggered by adding two classes to the animated component. The name of the
classes depend on the
transitionName
property. Assuming the
transitionName
property value
is
flash
, the classes are:
•
flash-appear
,
flash-appear-active
when the component appears;
