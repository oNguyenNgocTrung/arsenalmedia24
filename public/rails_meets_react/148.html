This example has really minimal output and only one dynamic feature - it is quite easy to test it.
Let’s see how we can test output, without testing stateful part of the component yet:
Example of testing static output of
Greeter
component.
Greeter =
React.createFactory(Components.Greeter)
describe
'Greeter component'
,
->
beforeEach
->
ReactTest =
React.addons.TestUtils
@greeter =
ReactTest.renderIntoDocument(Greeter(
initialName:
'my first test'
))
it
'has paragraph which greets someone'
,
->
html =
ReactDOM.findDOMNode(@greeter)
expect($(html).find(
'p'
).length).toEqual(
1
)
expect($(html).find(
'p'
).text()).toEqual(
"Hello, my first test!"
)
We are using React addon called
TestUtils
.
If you haven’t enabled addons in your config, please
go back to
Installation
chapter where it is discussed how to enable it.
.
renderIntoDocument
renders our component in
detached DOM node
- that means, you have your component rendered
but it is not displayed in your page.
In
beforeEach
we are rendering our component and making it accessible using instance variables.
You can use it in all
it
functions you define. Syntax is similar to RSpec.
In
it
block we’re calling
React.findDOMNode
to get real DOM rendered by React component. Then
we’re using jQuery selectors to fetch data we are interested to and check the output. As an exercise
