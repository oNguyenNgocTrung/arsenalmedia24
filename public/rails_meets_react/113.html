componentWillUpdate
This function is invoked
before
rendering when new props or state are being received - however,
not for the initial render.
You
can’t
use
setState
here - you should rather do this in
componentWillReceiveProps
if
needed.
How this one can be used? The function is called when you know that the component will be
updated, therefore - possibly moved, resized, etc.
componentWillUpdate
is the last time you can
memorize current low-level values of the HTML DOM node - like
height
or
scrollTop
. After that,
due to rerender, these values
might
be lost.
See an example inspired by
a blog post writen by Vjeux
.
Scroll doesn’t change because previous values are memorized in componentWillUpdate.
SimpleLog = React.createClass
componentWillUpdate: ->
node = React.findDOMNode(this)
@scrollHeight = node.scrollHeight
@scrollTop = node.scrollTop
componentDidUpdate: ->
node = React.findDOMNode(this)
node.scrollTop = @scrollTop + node.scrollHeight - @scrollHeight
getInitialState: ->
{ items: [] }
addEvent: ->
@setState items: [ 'Event ' + @state.items.length ].concat(@state.items)
componentDidMount: ->
@interval = setInterval(@addEvent, 300)
componentWillUnmount: ->
clearInterval(@interval)
render: ->
React.DOM.div
style:
height: '250px'
overflowY: 'scroll'
backgroundColor: '#bbb'
React.DOM.ul null,
@state.items.map((item) ->
React.DOM.li
key: item
item
