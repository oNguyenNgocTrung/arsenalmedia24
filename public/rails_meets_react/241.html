Upgrading React from 0.11 to 0.13
234
Transform your components to elements
Since starting from 0.13 passing a ‘bare’ component class to
React.render
is throwing an error,
we need to create an element from our component class before passing it to
React.render
. This is
done using
React.createElement
method:
Component = React.createClass
render: ->
# your render code
# this is a component class. It is NOT ready to create a renderable component from it.
element = React.createElement(Component, props, children)
# React returns a renderable element from your component class.
# You can pass this variable to React.render and your component will be rendered.
React.render(element, domNode)
You can pass a tag name (like
"div"
) or a component class as a first argument of
React.createElement
to create the React element from it. JSX is doing this transformation behind the scenes so you do
not need to do anything about your JSX syntax.
Calling
React.creatElement
each time you want to render a component of a given type (or used
within
render
function of an another component!) would be a huge and annoying task. Fortunately,
React provides us a great shorthand to it, called
React.createFactory
:
Component
=
React.createClass
render
: ->
#
your render code.
component
=
React.createFactory(Component)
#
component is now a
function
which takes props and children.
#
So it is the same syntax as Component(...) had
in
pre
0.12
era
!
React.render(component(props, children), domNode)
So basically you retain the same behavior as in 0.11 by wrapping your component class with a
factory. This is how
React.DOM
remained unchanged in 0.13 - behind the scenes it’s just an object
containing factories for all virtual DOM nodes! That’s our key to make a relatively easy upgrade!
Get rid of
getDOMNode
Starting from 0.13,
getDOMNode
is deprecated and is replaced by
React.findDOMNode(component)
.
You need to replace your
getDOMNode()
calls with
React.findDOMNode(@)
.
