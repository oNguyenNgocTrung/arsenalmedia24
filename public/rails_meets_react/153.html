Testing with references
In some cases you may not be interested in real HTML rendered by React. For example if you have
lower-level components tested with HTML output, you can find it redundant to test this HTML
output again in upper-level component’s test. This is where
refs
feature comes in handy - you
can test whether reference with given name and state/properties is defined. You can also test if
the reference is a component of valid type using
isDescriptorOfType
method inside
TestUtils
addon.
I personally use this approach often when testing some kind of edge cases like “list is empty” message
in list components. I declare such message as
listIsEmpty
reference and just set state to empty list
and check whether
listIsEmpty
ref is defined. I assume that I got a proper message for the user
inside this field. That allows me to change typos and message without changing tests with every
tiny change.
Let’s take more common example when this technique can be useful. Consider this invitation list. It
keeps list of invited people in root component’s state - all of them have ID and name.
