Foreword
We live in an interesting world for web developers.
Nowadays, the web browser is the most important part of typical user’s computer. Years passed,
and technology got more complex. What before was a simple program to request static pages on the
World Wide Web became a powerful tool to create rich media experiences. Audio, video, reactive
user interface. We tend to underestimate the importance of the frontend in our apps. Many of us
came from the backend world, where everything is so simple - you got a request, you render a reply.
That’s it.
But in the current world, building rich, powerful and visually appealing user interfaces
became a skill that every web developer must have
. Your client wants to have a good UI. Your
client demands it. And you need to meet your client’s expectations.
The Rails framework itself provides only basic help for dealing with challenges we may experience
during creating sophisticated frontend code. In the world of single-page applications like Facebook
or Twitter, web applications with tons of custom input solutions and dynamically changing data it’s
rarely helpful.
You may have came across many solutions to fix this problem. There are tons of JavaScript
frameworks. Angular or Ember are full-stack solutions which help us to create full-rich client-side
applications. They are big and often contain a lot of magic. They try to be a “one size fits all” solution.
Those kind of frameworks usually incorporate some architectural pattern like MVC. They have a
steep learning curve with brand new jargon
1
. They just don’t feel right if you want to stay with
simple code.
In Arkency we don’t like coupling with frameworks - we prefer using laser-focused libraries to get
the job done. It allows us to create maintainable and testable code in a short amount of time. It also
helps us to easily refactor when new features arrive. Just like in the backend. We researched and
mastered techniques and tools which help us to manage the complexity of our applications. The one
of the most recent and well-acclaimed tools in our toolkit is React.
React is the JavaScript library for building user interfaces. It was created by Facebook to make
frontend development faster, less error-prone and more maintainable. It is focused only on the view
part of the client-side application - rendering HTML or reacting to user generated events like clicks
or changing input fields. It’s like ERB for rich frontend solutions. React manages how DOM elements
render and behave on your website. Facebook have already been using it for a long time. If you are
a Facebook user you are using it every day. E.g. The comments box under your timeline elements is
rendered and managed by React.
