i18next - move your gettext to frontend
This approach was good for quick start but did not scale - it required multiple ajax calls to backend
on each page render, so we decided to find something better. After some research we have chosen
i18next
- full-featured i18n JS library that have pretty good compatibility with gettext (including
pluralization rules). With i18next you can easily return translations using almost the same API as
in gettext:
# {"key": "translation"}
i18n.t(
'key'
)
# => translation for key
