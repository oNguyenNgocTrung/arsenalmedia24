Testing
Testing is one of the most important practices widely used in Rails community. When I began
working with React I found it hard to find any good examples of how to test components. While
it may be tempting to go for full-stack acceptance testing, like testing through
Capybara
from the
backend side, you can test your components front-end only. Such tests are faster than capybara tests
since they do not need virtual browser to run your code.
I want to present you classical approach of testing component’s output using
React.findDOMNode
and external libraries like jQuery. First we’ll try to test by setting state directly. Then I’ll show an
improvement with simulating synthetic events.
In some circumstances you can even drop testing HTML, focusing only on testing reference presence
and/or state and properties. I generally favor this approach - since I’m confident given component
is providing good piece of HTML I can only focus on corectness on its state and properties.
We’ll start with very minimal example of the test. I’ll use
Jasmine
test framework in my examples.

Note about names availability in asset pipeline

names defined within a file are not available inside other files. While it may be convenient in many

using
