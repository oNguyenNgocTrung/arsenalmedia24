Testing through simulating events
Most of our components are made to be dynamic - and such dynamism comes typically from two
sources:
•
User interactions.
Users click on UI elements, drag elements or scroll the page. You can use
user’s built-in camera or microphone. This is the most common reason of component’s state
being changed.
•
Integration with external services.
You can have AJAX polling or push mechanism in your
client-side app which triggers component’s state change. Or you can have integration with
social media services, analytics / metrics tools and you query or you have events coming
from them. Your Rails backend is the ‘external world’ for your client-side app - apart from
being served by Sprockets it has no connection to it whatsoever.
React provides tools to test components through user interactions using simulated events. React
is using synthetic events under the hood so it is impossible to trigger events using, for example,
trigger
function in jQuery.
