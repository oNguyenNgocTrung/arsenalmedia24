componentWillUnmount
componentWillUnmount
is called right before the React component is
unmounted
. A React
component is unmounted when there is no real DOM for this component anymore. In this state,
you can not set the state of this component.
This method is usually used to do some kind of cleanup. You can clean intervals you set, dispose
3rd party event handlers, release resources you lock and so on.
componentWillUnmount
is the last
chance to access your state and properties.
Let’s take the cocktail example above. This implementation has a problem - your component might
unmount, but events would still be published. Since in this implementation
setState
is called after
an event is received, such situation will result in an error.
In the Bacon.js library which is used for providing an event bus, you can stop listening for changes
by calling the
end
method of the event bus instance. With
componentWillUnmount
you can call it
when your component dies.
Clean subscriptions before the component’s death.
CocktailList = React.createClass
displayName: 'CocktailList'
getInitialState: ->
cocktails: []
cocktailCategories: []
componentDidMount: ->
{eventBus} = @props
eventBus.onValue (event) =>
switch event.type
when 'cocktailsLoaded', 'cocktailsUpdated'
@setCocktails(event.cocktails)
when 'cocktailCategoriesLoaded', 'cocktailCategoriesUpdated'
@setCocktailCategories(event.cocktailCategories)
componentWillUnmount: ->
@props.eventBus.end()
