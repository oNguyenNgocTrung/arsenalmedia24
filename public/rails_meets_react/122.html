Passing state through references
Since React manages component’s life inside the library, there is no easy way to keep references
to lower-level components inside upper-level components. Keeping components inside instance
variables is a bad idea since even if you store this variable and update it after each render, you
never know that this particular instance is being mounted to your components tree. You can have
this confidence only if you are mounting through the
ReactDOM.render
method. Since we are
inside a component we are not using this method, thus we can’t use instance variables. Keeping
them in state is an anti-pattern since we shouldn’t store components in state (see ‘Anatomy of React
component’ chapter for more details) and it shares the same problem as with instance variables -
you can’t be sure your instance will be mounted. React can copy data from your instance and create
it by itself and this instance is not under your control anymore.
References is the way React solves this issue.
You can pass a special
ref
key to components
rendered inside upper-level component. The mounted instance is always accessible using the
@refs
object inside the component which owns the referenced component.
