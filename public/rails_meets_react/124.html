More
These are two main ways to pass state without using the third-party tools to make our components
more accessible from non-React-managed parts of the code. A) providing root component and B)
setting state through references (with still having top-level component acting as a facade).
An alternative solution is providing a publish-subscribe mechanism of any kind. We use RxJS
subjects and treat them as event streams. You can also use
Bacon.js
or any other library that provides
you a message bus you can pass to a React component to listen on data and publish outputs using
it. If you have such mechanism you can use
componentDidMount
and
componentWillUnmount
to
initialize subscriptions and unsubscribe when the component dies.
You can also split your UI into independent parts if possible - if you mount them separately using
ReactDOM.render
you will have more than one facade to operate with. If you don’t like the facade
idea,
@refs
is accessible also outside the root component. I’d rather advise against idea of calling
methods from outside world directly to references. It is an easy way to break the
law of demeter
and it makes your UI leaking abstraction. I rarely found it appropriate.
Passing state is still subject of our interest since we are unsatisfied with solutions we’ve tried. You
can also find a more proper way to pass state - classical approaches can be hard to operate with
when user interface becomes huge.
