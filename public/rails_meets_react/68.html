Setting initial state
Since state in components is
null
by default, it’d be hard to make something meaningful with React
if there was no way to have control the initial state. In fact, without such control you wouldn’t be able
to create any dynamic components at all. Since there will be only properties available and properties
are static, they shouldn’t change. As expected, React components provide you such control. You can
override the
getInitialState
method in our component to create the initial state that will be used
before any
setState
happens.
