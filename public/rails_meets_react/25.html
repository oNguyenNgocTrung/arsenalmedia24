Q&A
• Do I need to call
ReactDOM.render
only after
DOM is ready
(in the examples above
$
->
was used to detect this situation)?
To render with React inside the
<body>
or any other tag you need to make sure that such a
tag is already present on your page. You can do it in two ways. One way is to wait for the
proper event from the browser, for example by using jQuery, like in the examples above with
$ ->
.
Another way is to put your javascripts inside the
<body>
tag (usually at the end of it). In the
examples above, the first way is used since by default Rails places compiled assets in
<head>
so they are processed before the
<body>
is parsed by the browser.
• Can I build a virtual DOM using strings? Like
"<li>" + @state.foo + "</li>"
?
You shouldn’t do it because you can easily expose yourself to security bugs if the content is
not properly escaped. More about
dangerouslySetInnerHTML
