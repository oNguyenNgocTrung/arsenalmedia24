There is also a problem with (potentially) asynchronous behavior of setting state. Thus we are using
setTimeout
to register asynchronous function to be called. Since asynchronous functions work as
queue under the hood, we can be sure that this part of code will be evaluated after React performs
its updates.
Testing through simulating events adds more ‘acceptance’ feeling to testing your components. It
results in tests which can be used as ‘user interaction documentation’.
