Say hello to state
React has a way to distinguish what is
dynamic
from what is
static
. It introduces concept of
“static”
properties and
“dynamic”
state. The main trick of React is dead simple -
whenever your state
changes, your
render
method will be called again.
Why do we have this difference?
— you
may ask. There are performance reasons to do so — if you separate what’s static in a component
from what’s dynamic, React algorithms can handle this re-render in a very performant matter.
Facebook uses React to implement their most dynamic features like chat list and comment boxes.
Such elements must be performant!
We’ll enhance our component with a simple show/hide toggle — and you’ll quickly get how React
components handle dynamic changes. What we put in a state is some kind of toggle information —
a simple boolean whether it is hidden or not. Since you hide the whole list, choosing top component
(I’ll call it
root component
) seems a reasonable choice here. First of all, we’ll add a button to show
