It traverses all the components in tree and accumulate all components where a test is true. This is
not that useful on its own, but it’s used as a primitive for other test utils.
scry-prefixed functions
• scryRenderedDOMComponentsWithClass
• scryRenderedDOMComponentsWithTag
• scryRenderedComponentsWithType
Each returns an array of those components, whose
class
,
tag
or
type
matches the given argument.
The array could be empty.
the rest of find-prefixed functions
• findRenderedDOMComponentWithClass
• findRenderedDOMComponentWithTag
• findRenderedComponentWithType
They are named similarly to the above scry-prefixed ones, but:
• are prefixed
find
instead of
scry
• the
Component
in the middle is in a singular form
The behavior is also almost the same, in exception of:
• they expect there to be
exactly one
result (otherwise they throw an exception), and returns
that one result (instead of an array)
