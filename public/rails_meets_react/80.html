What is important, you should never call the

• If the state changes. It also calls

• If you mount component using

componentDidMount

The lifecycle methods such as the mentioned
componentDidMount

componentDidUpdate

be discussed later in the book.
An important assumption that React makes about your
render
method is that it is
idempotent
-
that means, calling it multiple times with the same properties and state will always result in the
same outcome.
