Controlled components allow you to make the input filtering before it is even displayed to the
user. You can replicate
errors
field functionality of ActiveRecord models on a component’s
level to provide the fastest possible validation output to your user. You can use it to validate
bad symbols in a field that requires special input format. With controlled components, you
get more control about what can go to the input field and how to provide feedback about the
input data. You can also see that controlled input components encourage us to store the state
on the higher level of your components tree — in this example, inside
UserForm
. You may see
later that it is quite a common pattern in React-managed user interfaces.
•
Uncontrolled components
— if you don’t pass any value (or pass
null
) as a
value
property
to
input
, it is called
an unmanaged component
. That means such input handles change like
a regular HTML input. It doesn’t care about react component state. You can still get its value
using
onChange
callback. Uncontrolled components are good when you don’t have input
filtering in this particular component. In such case, it is a bit simpler not to set state in
onChange
method.
I’d greatly recommend to rely on controlled components and treat them as your default option when
it comes to creating forms — they give you more control and even if you don’t use such controls yet,
maintaining controlled components is really easy - you just call
@setState
at the end of
onChange
with new state. That’s it!
Visit
this website
to see how to read new state based from handler event data for every kind of an
input component.
