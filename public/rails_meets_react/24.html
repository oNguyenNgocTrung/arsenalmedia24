The initial

{clicked:
false}
. When you click the

true}

again and this time the
render()

React analyzes the difference between what was previously rendered and what is rendered
now
. Based on that it removes the
<a>
tag from the real
<div>
and inserts a
<span>
tag instead.
You no longer need to manually use
ReactDOM.render
to propagate changes. When you use
@setState
, React schedules re-rendering of a component for you. It handles deletions, updates
and inserts to the DOM structure. And it is optimized to do it as fast as possible.
That’s a very simple component usage. But it illustrates the basic blocks of how React works:
• virtual DOM for specifying what should be rendered
• reading and updating state based on events
• displaying different content based on the state of the component
• mounting the element with
ReactDOM.render
• composing bigger components from smaller components
