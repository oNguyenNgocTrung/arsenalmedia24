Mastering Components
Components - the main building block
React is a focused library which has small mental overhead when it comes to learn its “language”.
The main building block that is used in React is called a
component
.
Component is the part of the HTML rendered and managed by React. It is made by state
and properties and always composed of smaller React components. React comes with built-in
components for all basic HTML5 tags which allows you to build your own components on top
of them. A component is a stand-alone and self-contained part of the UI code. You can reuse
it anywhere you want and use it to compose bigger components.
Apart from helpers and some more sophisticated tools that React provides, this library is all about
components. User interface you’re creating will be
composed of
components you create. The art
of programming in React is creating components from smaller ones to achieve your user interface
goals. You may find it easier to think about it as a
partial
in your typical Rails ERB view. The main
difference is that views made from partials are typically shallow in structure - I find it really rare
to have more than three level of depth in Rails partials. On the other hand, React encourages you to
create complex tree structure from our components.
