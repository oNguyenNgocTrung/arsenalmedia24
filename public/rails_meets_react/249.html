Disclaimer: Do not wrap your createClass in
createFactory!
The general rule of thumb when working with factories is to create them on the
consumer side
. You
may be tempted to do something like this:
Component = React.createFactory(React.createClass(
# ...
))
Don’t do it.
It makes your code unable to be used with JSX and tested with Facebook provided testing
framework called
Jest
. It is considered an anti-pattern. Of course if this features are not going to be
used you can make this decision - but consider these consequences first.
Better way: Leverage CommonJS to simplify
wrap-with-factory process
If you are using CommonJS, you can easily perform wrapping-with-factory step of React upgrade.
You can use
Browserify-Rails
or
Sprockets-CommonJS
gems to easily include CommonJS module
system to your Rails app. I’ll make an example based on the Sprockets-CommonJS approach.
The first step is to place your component classes in a separate, CommonJS-managed files. In the code
above you can extract 5 files:
•
month_name.module.js.coffee
•
form_input_with_label.module.js.coffee
•
form_input_with_label_and_reset.module.js.coffee
•
date_with_label.module.js.coffee
•
separator.module.js.coffee
Then just copy-paste component classes, adding
module.exports = <ComponentName>
at the end
of each file.
