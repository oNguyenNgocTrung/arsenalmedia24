As a consequence, all data needed to display and/or create lower components and not coming from
an user will be stored in top-level’s component state. This architecture pattern we often call
root
component
. If you design your component this way you may find that you end with root component
being stateful and having relatively big state data and all lower-level components stateless. It is
natural with this kind of design. Also all kind of event handlers will go to the upper layer with their
implementation, since events often change state. Stateless components below should have a property
for binding to their browser events, or binding to custom events you write yourself. Typically such
custom events will be called inside the browser event handlers code, though.
