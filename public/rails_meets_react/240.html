Upgrading React from 0.11 to 0.13
Recently React got an update which is a step forward towards making your components less “magic”.
There are some breaking changes, especially if you are not using JSX. Fortunately, upgrade process
is rather straightforward.
API changes:
Here are the method calls from the API you need to find and take an action:
•
React.renderComponent
(rename to
React.render
)
•
React.renderComponentToString
(rename to
React.renderToString
)
•
React.renderComponentToStaticMarkup
(rename to
React.renderToStaticMarkup
)
•
React.isValidComponent
(rename to
React.isValidElement
)
•
React.PropTypes.component
(rename to
React.PropTypes.element
)
•
React.PropTypes.renderable
(rename to
React.PropTypes.node
)
•
React.isValidClass
(deprecated; remove it from your codebase)
•
transferPropsTo
(deprecated; remove it from your codebase)
Upgrade to 0.12 first
Every time React introduces major breaking changes there is a transitional period of one version to
make sure you can gradually upgrade your code. This is also the case with 0.11 -> 0.13 upgrade -
0.12 introduced all major changes while preserving the old API of React. Also, deprecation warnings
were introduced so you’ll know what won’t work in 0.13. That way you can upgrade your code by
looking for warnings and fixing related pieces of code.
This is a relatively safe step and I greatly recommend to do it.
