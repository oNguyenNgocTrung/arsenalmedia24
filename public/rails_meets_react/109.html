componentWillMount
This function is invoked only once,
before
mounting a component (therefore, before
componentDid-
Mount
). There could be no real DOM yet. However, the function is invoked
after
getInitialState
.
componentWillMount
is an appropriate place to call
setState
for the first time in your component,
but most likely you want to define the state in
getInitialState
instead.
To see a real-life usage of
componentWillMount
, consider the
debounce
method.
debounce
transforms a function into a throttled one. It cannot be invoked if the previous call happened earlier
than a certain amount of time. In other words - if a function in a component should not be called
too often
, it can be
throttled
by the
debounce
method.
The details about
debounce
are not that important right now. The point is -
we need a separate
throttled function per component instance.
.
Find out more about using debounce in React.js components on Stack Overflow
In addition, we want to throttle a method before even mounting the component, because it is used
in
render
. Therefore, we should do inside
componentWillMount
.
We will use
debounce
implementation from lodash.
