Event Handlers
Creating handlers for certain user events like clicks is a well-established pattern in the JavaScript
world. React is not re-inventing this pattern - you can use just plain handlers to work with user
generated events.
No surprises
React does not provide new types of events - it only supports well-known browser events you
know.
That means you do not need to learn new events, or change browser events’ naming - it stays
the same. You can use
this reference
or follow list of events React supports
here
.
Event handler defining convention
In DOM components provided by React inside
React.DOM
module there are special properties to
set event handlers. They are easy to recognise - all begin with an
on*
prefix.
Synthetic events
There are subtle differences between browsers when it comes to event handling. Fortunately, you
don’t need to worry about it when you are working with React. It ensures that all events behave
identically in IE8 and above by implementing a synthetic event system. Synthetic events are thin
wrappers to actual browser’s events. They adhere to original event API so you shouldn’t have
problems if you worked with classic event handlers before.
You can use
preventDefault
on a synthetic event just like you would in a typical browser event.
